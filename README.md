# Moscow Geocoder

Сервис геокодирования московских адресов. Решение нормализует входной адрес, сопоставляет его со справочником зданий и возвращает координаты и ранжированный список совпадений.

## Задача
Нужно по свободному текстовому адресу найти соответствующий объект в справочнике, даже если адрес содержит сокращения, опечатки или неполную запись улицы и номера дома.

## Архитектура решения
```text
Входной адрес
      │
      ▼
Нормализация текста
      │
      ▼
Точное сопоставление
      │
      ├──► если найдено
      │
      ▼
Нечеткое сопоставление
      │
      ▼
Ранжирование кандидатов
      │
      ▼
Координаты и список совпадений
      │
      └──► API FastAPI
```

## Структура репозитория
- `src/data_loader.py` — загрузка справочника зданий;
- `src/normalize.py` — нормализация полей адреса;
- `src/geocode_basic.py` — базовый контур сопоставления;
- `src/geocode_improved.py` — улучшенный контур сопоставления;
- `src/api.py` — API;
- `scripts/run_basic_cli.py` — запуск базового контура;
- `scripts/run_improved_cli.py` — запуск улучшенного контура;
- `scripts/run_evaluate.py` — сценарий оценки;
- `docker-compose.yml` — запуск сервиса в контейнерах.

## Метрики
- доля корректно найденных адресов;
- качество текстового совпадения;
- географическая ошибка;
- доля попаданий в верхние позиции списка кандидатов;
- задержка ответа API.

## Результаты
Численные результаты должны быть пересчитаны сценариями оценки и сохранены в `reports/metrics_summary.md` и `reports/metrics_summary.json`. В README необходимо указать:
- качество базового и улучшенного контура;
- уменьшение географической ошибки;
- изменение доли корректных попаданий.

Формат записи:
- `Улучшенный контур повысил долю корректных совпадений с ... до ...`
- `Средняя географическая ошибка сократилась с ... до ...`

## Запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn src.api:app --host 0.0.0.0 --port 8000
```

### Запуск в Docker
```bash
docker compose up --build
```

## Ограничения и развитие
- Качество зависит от полноты и чистоты адресного справочника;
- сложные случаи с неоднозначными сокращениями требуют отдельной логики;
- следующий шаг — добавить отчёт по ошибкам и типовым сценариям отказа.
